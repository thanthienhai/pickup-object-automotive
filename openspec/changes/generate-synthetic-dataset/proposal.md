## Why

Now that we have successfully extracted individual object patches using YOLO detection, we need to artificially expand our dataset to make the model robust against different environments. By pasting these extracted object patches onto various background images, we can automatically generate thousands of new training examples. Crucially, this script needs to auto-generate the corresponding YOLO format `.txt` label files based on the filename of the pasted objects, eliminating the need for manual annotation.

## What Changes

- Create a new Python script `create-data-v1/generate_dataset.py`.
- Implement a pipeline that randomly selects a background image and an extracted object patch.
- Paste the object patch onto the background at a random position, scale, or rotation.
- Automatically derive the class label from the object's filename prefix (e.g., `bong-ban-1.png` becomes `bong-ban`).
- Generate the corresponding YOLO format bounding box label file (`.txt`) for the newly created synthetic image.
- Output the generated images and labels into a structure perfectly matching the YOLO `dataset.yaml` format (`images/train`, `labels/train`).

## Capabilities

### New Capabilities
- `synthetic-dataset-generator`: A capability to automatically overlay object patches onto background images, dynamically assign class IDs based on filenames, and output a ready-to-train YOLO dataset structure.
